Promise và xử lý bất đồng bộ trong JavaScript

1. Lập trình bất đồng bộ là gì?#

Bất đồng bộ giúp chương trình không bị chặn khi xử lý các tác vụ mất nhiều thời gian như gọi API hoặc đọc dữ liệu.

2. Promise#

Promise đại diện cho một giá trị sẽ có trong tương lai, có 3 trạng thái:

  • Pending
  • Fulfilled
  • Rejected

3. async / await#

async / await giúp viết code bất đồng bộ dễ đọc và dễ bảo trì hơn.

4. Ứng dụng thực tế#

Xử lý bất đồng bộ thường được dùng khi gọi REST API trong các ứng dụng Web hiện đại.
